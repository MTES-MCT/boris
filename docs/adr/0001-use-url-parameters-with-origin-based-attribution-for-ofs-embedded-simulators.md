# Use URL parameters with origin-based attribution for OFS embedded simulators

OFS embedded simulators use a dedicated embed surface that reuses the current Boris eligibility simulator internals. OFS websites can customize presentation and workflow through documented URL parameters, while Boris controls attribution through admin-managed `ofs_embed_origin` records that map authorized website origins to OFS records; API writes derive the source OFS from the authorized origin and store only the source type and source OFS on the simulation. We chose this over server-side embed configuration objects so OFS integrations remain easy to tweak without Boris-side configuration for every presentation change, while still preventing URL parameters from deciding which OFS receives contact lines. Browser embeds cannot safely hold an API key, so embed API writes use origin authorization to prevent unauthorized website embedding, with rate limiting and payload validation mitigating abuse from non-browser scripts.
